Exercise 4 : Stats with Kafka

Previously on HOMicS -> Exercise 3: Stats

Context

The previous solution has a considerable drawback, we have to add logic in the monolith and it's not really scallable.

Goal

Replace this manual approch by using kafka. All the kafka config is already done for you. it's contained in the module common-messaging.

common-lib -> config kafka

add the dependency to your pom:

<!--All dependencies for kafka and messaging-->
<dependency>
    <groupId>com.homics.commons-messaging</groupId>
    <artifactId>commons-messaging</artifactId>
    <version>1.0.0</version>
</dependency>

To send a message with kafka :

  private KafkaTemplate<String, OrderPayedMessage> kafkaTemplate;

  Message<OrderPayedMessage> message = MessageBuilder
                .withPayload(new OrderPayedMessage(1,1,"user"))
                .setHeader(KafkaHeaders.TOPIC, TOPIC_STATS)
                .build();

        kafkaTemplate.send(message);

To retrive a message with kafka :

   @KafkaListener(topics = TOPIC_STATS, groupId = GROUP_ID, containerFactory = "statsMessageFactory")
   public void onImpactStockMessage(@Payload OrderPayedMessage impactStockMessage) {

    }

Steps

  1. Checkout on exo4-kafka on the stat repository.
  2. Listen to kafka and retrieve OrderPayedMessages.
  3. Save this message on the database
  4. On the monolith send OrderPayedMessages.
  5. Remove StatsTask and OrderStat table and the related code.

Verification

To verify that stat-activity is well implemented, launch th monolith, pass some orders. Then start the micro service and verify that the stats are here.
